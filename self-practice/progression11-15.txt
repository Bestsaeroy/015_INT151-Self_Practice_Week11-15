Week 11
- ฝึกสร้างหน้าเว็บ SelfPractice แบบง่าย ๆ ด้วย HTML และใช้ JavaScript จับ DOM + event พื้นฐาน 
(เช่น กดปุ่มเพื่อเปลี่ยนรายละเอียดและไฮไลต์รายการ) ทำให้เริ่มคุ้นกับการเชื่อม “ข้อมูลในโค้ด” เข้ากับ “สิ่งที่เห็นบนหน้าเว็บ”

Week 12
- ฝึกจัดการ state ฝั่ง frontend ผ่านฟอร์มลงทะเบียน โดยเก็บข้อมูลใน array ของ object (registrations), 
ทำ validation (กันกรอกไม่ครบ/กันลงซ้ำ) และ render รายการลงทะเบียนใหม่ทุกครั้งที่ state เปลี่ยน ทำให้เข้าใจแนวคิดว่าหน้าจอควรอัปเดตตามข้อมูลปัจจุบัน

Week 13
- ฝึกดึงข้อมูลจาก “mock backend” ด้วย fetch() จากไฟล์ workshops.json พร้อม flow การโหลดที่มีทั้งข้อความ Loading 
และการจัดการ error เมื่อโหลดไม่สำเร็จ ทำให้เห็นภาพการทำงานแบบ frontend ที่คุยกับแหล่งข้อมูลภายนอกแทนการเขียนข้อมูลแข็ง ๆ ในโค้ด

Week 14
- ฝึกออกแบบ UX flow ที่ซับซ้อนขึ้น เช่น การยกเลิกการลงทะเบียนที่ต้องมี dialog ยืนยัน (สร้างเองด้วย HTML+JS) และลองจำลอง error จาก 
fakeCancelApi() (เช่น ยกเลิกไม่ได้เพราะ workshop ผ่านไปแล้ว) เพื่อแยก business logic ออกจากการแสดงผลให้ผู้ใช้

Week 15
- ฝึกใช้ JavaScript Date/Time แบบจริงจัง ทั้งการแปลง ISO string เป็น Date, การใช้ toLocaleString แสดงเวลาใน timezone Asia/Bangkok 
และการใช้เวลาเป็นเงื่อนไขของระบบ เช่น ตรวจว่าตอนนี้อยู่ในช่วง Registration Window หรือไม่ และใช้เวลาเริ่ม–จบของ workshop 
เพื่อกำหนดสถานะ ENDED / IN PROGRESS / STARTING SOON ทำให้เห็นว่าระบบสามารถตอบสนองตาม “เวลา” ได้ ไม่ใช่แค่ตาม action ของผู้ใช้อย่างเดียว
